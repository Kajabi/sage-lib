# Button

Buttons are used to perform actions within a page or view.

## Usage Guidelines

### When to use

- Communicating an action that will occur. (e.g. saving a page)
- Triggering or enabling an action, such as submitting requested information. (e.g. forms)
- Progressing or regressing a user through a step in a flow. (e.g. modals)

### When not to use

- Directing users to a new page or different part within the same page. Instead, use [Link](/docs/sage-link--primary).

## Accessibility

When following our usage guidelines, the button should be accessible right out of the box but there are still a few general accessibility best practices to make sure you follow:

- Buttons should always contain helpful text about what they do even if this text is expected to be visually hidden. (e.g. "Submit Form" or "Delete Account")
- Buttons should not disable tabbing or keyboard navigation.

## Additional Properties

### Align End
Floats a button to the right of the container. It's default is set to `false`.

<Canvas>
  <Story name="Align End">
    <Button alignEnd={true}>Button</Button>
  </Story>
</Canvas>

### Children
Sets the text of the string that displays in the button.

<Canvas>
  <Story name="Button Text">
    <Button>Button Text String</Button>
  </Story>
</Canvas>

### Class name
Assigns a class name to the `button` for style customizations.

<Canvas>
  <Story name="Class name">
    <Button className="your-class-name-here">Button</Button>
  </Story>
</Canvas>

### Custom Content
Allows you to apply a custom CSS class when you provide custom HTML content within the button. The primary purpose of using this custom CSS class should be to manage the layout of the custom content within the component.

<Canvas>
  <Story name="Custom Content">
  <Button
    color="secondary"
    hasCustomContent={true}
    customContentClassName="demo-custom-class">
    <Card>
      <Card.Block>
        <p>This is content in a card, in a button</p>
      </Card.Block>
    </Card>
  </Button>
  </Story>
</Canvas>

### Link Tag
Allows you to pass a `Link` for React Router Links.

<Canvas>
  <Story name="Link Tag">
    <Button linkTag="React Router Link goes here">Get Started</Button>
  </Story>
</Canvas>

### onClick
Allows you to pass a function to trigger when the button is selected.

<Canvas>
  <Story name="onClick">
    <Button onClick={() => alert("Great success!")}>I trigger an alert!</Button>
  </Story>
</Canvas>
